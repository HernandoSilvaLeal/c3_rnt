PLAN DE TRABAJO – DESARROLLO SITIO WEB
Cliente: Red Nacional de Transportes (RNT) Proveedor: AppCors S.A.S. — NIT 901 824 058 Responsable: Ing. Hernando Silva Leal Fecha: Bogotá, martes 16 de septiembre de 2025
1) Antecedentes y objetivo
El cliente dispone actualmente de un sitio web con un cotizador auxiliar implementado en tecnologías antiguas. Se requiere modernizar el sitio y el cotizador, creando una versión estática con HTML, CSS y JavaScript (no se usará Angular ni React), con excelente presentación final y alineación de imagen corporativa.
Objetivo general:
• Identidad visual corporativa completa.
• Cotizador para cliente final (tarifario cargado y actualizado: mensajería, paquetes y paqueteo, según indicaciones del cliente).
• Rastreo por número de guía conectado al software actual del cliente (GEL OTRA) e integración con API de la empresa EXCE cuando corresponda.
• Autenticación (login) para secciones que lo requieran.
• Integración con WhatsApp y redes sociales.
• Manual de operario y despliegue en el dominio existente del cliente (rednacionaldetransportes.com).
Kick-off: realizado el miércoles 10 de septiembre de 2025 (7:30 p. m., Google Meet, duración: 1 hora). De esta reunión se consolidaron los requerimientos que aquí se formalizan.
2) Alcance por fases, entregables y criterios de aceptación
Fase 1 – Diseño e Imagen Corporativa
Alcance:
• Maquetación base (HTML/CSS) del sitio estático.
• Aplicación de imagen corporativa: paleta, tipografías, logotipo, estilos de componentes y lineamientos UI.
• Páginas públicas acordadas (inicio/portada, servicios, contacto, etc.).
Entregables:
• Prototipo navegable en entorno de pruebas.
• Guía de estilos breve (fuentes, colores, componentes).
Criterios de aceptación:
• Aprobación del diseño visual por parte del cliente.
• Sitio base navegable y consistente con la identidad corporativa.
Fase 2 – Funcionalidades: Cotizador + Rastreo de Guía
Alcance (cotizador):
• Implementación del tarifario (mensajería/paquetes/paqueteo) como calculadora en JavaScript.
• Carga del tarifario según documentos entregados por el cliente; reglas/zonas/variables tal como se indiquen.
Alcance (rastreo):
• Rastreo por número de guía consultando el software actual del cliente (GEL OTRA) y presentando estados de la guía.
• Automatización de la consulta de estado; cuando corresponda, integración vía API de EXCE (cuando sea disponibilizada).
Entregables:
• Módulo Cotizador funcional con validaciones.
• Módulo Rastreo funcional con búsqueda por guía y visualización de eventos/estados.
Criterios de aceptación:
• Cotizador calcula correctamente los valores definidos en el tarifario oficial proporcionado por el cliente.
• Rastreo trae y muestra estados de guía conforme al sistema actual del cliente/EXXA (cuando la API esté disponible).
Fase 3 – Integraciones finales, despliegue y manual
Alcance:
• Botón de WhatsApp automatizado y vinculación a redes sociales del cliente.
• Autenticación (login) para secciones restringidas (en sitio estático, usando la mecánica definida con el cliente).
• Despliegue a ambiente productivo en el dominio existente rednacionaldetransportes.com.
• Manual de operario (uso del cotizador, rastreo, contenidos y lineamientos básicos).
Entregables:
• Sitio publicado en producción.
• Manual de operario (PDF).
• Entrega de usuarios/contraseñas y/o instrucciones de acceso.
Criterios de aceptación:
• WhatsApp y redes sociales operativas desde el sitio.
• Sitio en línea en el dominio del cliente (rednacionaldetransportes.com), con manual entregado.
3) Reuniones y coordinación
• Reuniones de avance/entrega: los días martes por Google Meet.
• Acceso a dominio: se requerirán un par de reuniones técnicas para obtener y validar accesos al dominio actual (rednacionaldetransportes.com), DNS y/o hosting necesarios para el despliegue.
• Se entregarán avances iterativos semanales hasta completar cada fase.
4) Dependencias y suministros del cliente
• Tarifario oficial y actualizado (formatos y reglas necesarias) para el cotizador.
• Accesos/credenciales y/o endpoints para el sistema actual de rastreo y, cuando aplique, API de EXCE.
• Accesos al dominio/hosting para el despliegue en producción (rednacionaldetransportes.com).
• Lineamientos finales de imagen corporativa (logotipos, paleta, etc.).
• Nota: El sitio será estático con JavaScript según lo definido. No se utilizarán frameworks SPA (no Angular ni React).
